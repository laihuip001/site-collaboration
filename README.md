# 現場連携プロジェクト (Template)

設計（オフィス）と現場（フィールド）の情報共有用リポジトリテンプレートです。
「言った言わない」を排除し、常に最新の図面を正解とするための構造になっています。

> [!NOTE]
> **💡 用語についての補足**
>
> 「設計」「現場」「図面」などの用語は**コード/システム開発プロジェクトにおける役割の比喩**です。
>
> - **設計担当** → システムアーキテクト / CEO
> - **現場監督** → 実装エンジニア / Constructor
> - **図面** → 設計ドキュメント / 仕様書

## 📂 ディレクトリ構成

```text
/
├── docs/
│   ├── current_blueprints.md  # 👈 現場はここだけを見る（最新図面リンク集）
│   ├── change_log.md          # 変更履歴の要約
│   └── decisions/             # 重要な決定事項の記録 (ADR)
└── README.md
```

## 🚀 運用ルール

### 1. 現場監督のルーチン

* 図面確認は必ず `docs/current_blueprints.md` を開く（スマホ対応）。
- Google Driveのフォルダを直接漁らないこと（古いファイルを掴むリスクがあるため）。
- 不明点はGitHub Issueで質問する（LINE/電話は緊急時のみ）。

### 2. 設計担当のルーチン

* Google Driveに修正図面をアップロードしたら、即座に `current_blueprints.md` のリンクを更新する。
- 重要な決定は口頭で終わらせず、Issueに記録するか `decisions/` にドキュメントを残す。

## 📥 セットアップ手順

1. このフォルダ構成をGitHubリポジトリにPushする。
2. Google Driveに「Release（現場公開用）」フォルダを作成する。
3. 現場監督のスマホにGitHubアプリをインストールする。
